Privacy & Security Analysis

This page outlines the key security and privacy considerations specific to this Mobile Inventory Application.

1. Input Validation & XSS Prevention

Consideration: Malicious users might try to enter harmful scripts (Cross-Site Scripting) into fields like "Item Name" or "Comment".
Implementation: Angular automatically sanitizes all values bound to the HTML (using interpolation {{ '{{"{{"}}' }}). Additionally, we implemented strict form validation to ensure quantities and prices cannot be negative.

2. Data Persistence & Local Privacy

Consideration: Inventory data might contain sensitive supplier information or pricing strategies.
Implementation: Currently, this app runs entirely on the client-side (Session-based). Data is not sent to any external server, meaning privacy is maintained locally. However, closing the browser will clear the data. In a real-world mobile app, secure encrypted local storage (like SQLite) would be required.

3. Authentication & Authorization (Future Scope)

Consideration: Unrestricted access allows anyone holding the device to delete or modify the inventory.
Implementation: To make this application production-ready, a secure login system (using JWT or OAuth2) must be integrated. Different roles (e.g., Admin vs. Staff) should be created so only authorized personnel can execute the "Delete" function.
