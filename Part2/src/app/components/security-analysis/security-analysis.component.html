Privacy & Security Analysis

To achieve a secure mobile environment, this application addresses specific privacy and security vulnerabilities intrinsic to standalone web and mobile systems.

1. Input Validation & XSS Prevention (Security)

Vulnerability: Malicious inputs (e.g., typing scripts instead of Item Names) can lead to Cross-Site Scripting (XSS) attacks.
Solution: The application rigorously validates inputs. Numeric fields are restricted at the keystroke level to reject letters. Furthermore, Angular's default data-binding strictly sanitizes outputs, automatically neutralizing malicious HTML/JS payloads.

2. Data Persistence & Local Privacy (Privacy)

Vulnerability: Storing inventory details on a mobile device without server sync risks exposing business data if the device is lost.
Solution: Currently, data is maintained strictly within the browser's volatile Session Memory using Angular Services. It leaves no footprint once the application closes, protecting supplier and pricing privacy. In a production mobile environment, data would be stored using encrypted SQLite to comply with the Australian Privacy Principles (APP 11: Security of Personal Information).

3. Middleware & Architecture Considerations

Analysis: Because mobile devices often switch between network architectures (Wi-Fi/4G/5G), relying solely on server-side validation is unsafe.
Solution: This standalone app uses a "Client-heavy" architecture, ensuring that all logic, validation, and state management occur securely on the frontend via TypeScript, maintaining data integrity even offline.
